Tweet Verification Assistant

Get help in analyzing the veracity of a tweet

Description of features

Text length

The length in characters of the tweet text. The tweet text is first preprocessed by removing HTTP links, hashtags, user mentions and retweets.

Number of words

The number of words in the tweet text. The tweet text is first preprocessed by removing http links, hashtags, user mentions and retweets. Additionally, several useless symbols are removed such as !,=,< etc.

Number of positive/negative sentiment words

We support three different languages English, Spanish and German, for each of which we make use of a different list of sentiment words. For English we use the list provided by Jeffrey Breen, for Spanish the Spanish adaptation for ANEW and for German the Leipzig Affective Norms for German. For other languages, no such feature is computed.

Number of slang words

For slang detection we use predefined lists of slang words in English and Spanish. For other languages no such feature is computed.

Readability

An open-source implementation of the Flesch Reading Ease method is used to compute a readability score in the range 0-100 [0: hard-to-read, 100: easy-to- read].

Web metrics

Depending on the existence of external URLs, the tweet reliability is quantified based on a set of Web metrics. For tweet based features, external URLs found in the tweet text are examined while for used-based features, the URLs found in the Twitter profile description.

  • WOT trust: The Web of Trust (WOT) score which is a way to assess the trust on a website using crowdsourced reputation ratings.
  • Indegree Ranking/ Harmonic Centrality: The in-degree and harmonic centralities of the Web graph are looked up based on the rankings computed by the Web Data Commons project.
  • Four Alexa metrics (rank, popularity, delta rank and reach rank) based on the rankings API.
